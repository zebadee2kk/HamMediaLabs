# Voice Authenticity System

## Purpose
HamMediaLabs must avoid generic AI slop. Each brand should sound like a specific human with opinions, history, rhythm, taste, and limits.

This system converts brand voice from loose adjectives into operational material that agents, editors, and reviewers can use.

## Core principle
Prompts alone do not create human-feeling brands. Deliberate voice systems do.

## Required per brand
Each brand must maintain a `voice.md` file containing:

### 1. Character bio
A 1-2 paragraph persona biography. The persona should feel like a real operator, not a vague audience archetype.

Include:
- Background
- Frustrations
- What they have tried
- What they now believe
- What they refuse to tolerate

### 2. Voice contrasts
Define the brand in tensions.

Examples:
- Warm but not saccharine
- Confident but self-deprecating
- Direct but not rude
- Skeptical but not cynical
- Funny but not cruel

### 3. Voice DNA
Document:
- Favourite phrases
- Pet peeves
- Sentence rhythm
- Slang / jargon level
- Cultural references
- How the persona handles mistakes
- How the persona handles hype

### 4. House examples
Every brand needs 5-10 gold-standard examples:
- Strong opening
- Mid-piece rant
- Measured recommendation
- Takedown of bad advice
- Comment-reply style
- Short-form social post
- Long-form intro
- Bad example to avoid

### 5. Anti-voice
Explicitly document what the brand must never sound like.

Common banned AI patterns:
- "In today's fast-paced world"
- Bland listicles without a strong point of view
- Unnatural positivity
- Repetitive paragraph structures
- Hedging without reason
- Over-explaining obvious points
- Generic motivational filler

## Prompt and router guidance
Generation should use:
1. Persona bio
2. Brand voice file
3. Lab style guide
4. Recent approved examples
5. Target audience
6. Secure prompt skeleton

Recommended pattern:
- First draft: higher creativity
- Persona critique: what would this persona hate about the draft?
- Revision: tighten voice, remove AI tells, increase specificity
- QA: human review before publishing

## Human quality gates
Human review remains mandatory.

Reviewers should:
- Read sample sections aloud
- Ask whether the persona would actually say this
- Remove generic AI tells
- Add specific lived detail where needed
- Preserve useful imperfection and rhythm

## Specificity rule
At least one specific, concrete detail should appear in most long-form pieces. This can be:
- A messy operational anecdote
- A tool failure
- A cost comparison
- A real trade-off
- A practical mistake

Never invent false personal experiences. If an anecdote is illustrative or composite, label it honestly.

## Voice refresh cadence
Every 3-6 months:
- Review top-performing content
- Extract winning phrases and structures
- Add examples to the voice library
- Remove stale voice rules

## Metrics
Track proxies for human feel:
- Time on page
- Comments
- Shares
- Replies
- Repeat visitors
- Low bounce on long-form pages

## Bottom line
AI provides speed and variation. Human taste, judgment, and iterative editing create defensible brand personality.

# Account Naming Convention

## Purpose
Create consistent, scalable parent-child identity structure.

## Parent layer
### Format:
hammedialabs.[function]

### Examples:
- hammedialabs.master
- hammedialabs.ops
- hammedialabs.finance
- hammedialabs.security

## Brand layer
### Format:
brandname.[function]

### Examples:
- aiescape.editorial
- aiescape.social
- corp-satire.ops
- ukescape.analytics

## Disposable/testing
### Format:
lab-[provider]-[purpose]

## Rules
- Never mix parent and brand credentials
- Separate social identities where possible
- Separate analytics views
- Standardize for automation parsing

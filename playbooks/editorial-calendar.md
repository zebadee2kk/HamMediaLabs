# Editorial Calendar (per-brand operating rhythm)

> Content pillars existed only as bullets in the X strategy file;
> cadence targets only inside each `profile.md`. This playbook turns
> both into a weekly operating rhythm the operator can actually run —
> and hand over. Slots are commitments to *cadence*, not to topics:
> topics come from briefs (`trend-to-brief`, cornerstone plans,
> `source_signal` review).

## The weekly grid (MVP, one operator)

| Day | Brand A (SEO + X) | Brand B (TikTok + IG mirror) | Brand C (SEO + Reddit) |
|---|---|---|---|
| Mon | X post (from last week's piece) | — | Reddit comment-first pass (no links) |
| Tue | **Publish day** — cornerstone/secondary + newsletter | Clip 1 ships | X mirror post |
| Wed | X post | — | **Publish day** — piece + newsletter |
| Thu | X post (the deliberately contrarian one) | Clip 2 ships | Reddit comment-first pass |
| Fri | X post | Clip 3 ships (weekly ceiling: 3–5) | X mirror post |
| Sat | Weekly review (`playbooks/weekly-review.md`) — all brands | ← | ← |
| Sun | — | — | — |

Ceilings are governance, not targets: X per `docs/x-platform-risk.md`
§7, TikTok 3–5/week per Brand B profile, Reddit ≤1 post/sub/week
(comment-first) per master plan §3.3. **Hitting the ceiling is never
the goal; hitting the quality bar at a sustainable cadence is.**

## Pillar rotation (what fills the publish slots)

Rotate so no pillar goes dark for a month:

- **Brand A** (pillars per `profile.md` / X playbook): workflow
  deep-dive → stack/cost piece → "what I dropped" retro → niche-role
  workflow. One deliberately contrarian X thread per week.
- **Brand B** (series per `voice.md` §"Series candidates"): rotate
  Translations / Calendar reads / Slack history / 1:1 bingo; The
  Framework and one-shots fill the third slot. Never two instalments
  of the same series in one week (audience fatigue reads as the
  account running dry).
- **Brand C**: price-cap/news-peg when there's a peg, otherwise
  cornerstone-supporting explainer → regional/case-study piece →
  "the maths" breakdown. Quarterly Ofgem announcements pre-block a
  publish slot the week they land.

## Planning cadence

- **Weekly (in the Sat review):** confirm next week's slots have briefs
  attached (`content_asset.status = 'brief'` or better). An empty
  Tuesday slot on Saturday = the Monday is for brief-writing, not
  drafting.
- **Monthly:** pillar-balance check (any pillar dark >4 weeks?), kill
  or refresh underperforming recurring formats, pre-block known pegs
  (price-cap dates, platform events).
- **Quarterly:** voice refresh (`playbooks/voice-refresh.md`) +
  cornerstone audit (keep / refresh / retire per master plan §8.2).

## Rules

- A slot with no QA-passed piece **slips silently** — publishing an
  unpassed piece to save the calendar is the one trade this system
  never makes.
- Newsletter ships on publish day or not at all that week (no
  standalone filler issues at MVP).
- The calendar is repo state, not memory: next week's plan lives in the
  weekly review notes; the pipeline itself is visible on the dashboard
  (content-pipeline table).

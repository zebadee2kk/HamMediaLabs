# Brand A — Cornerstone Content Briefs (first 5)

> Each brief is a contract with the writer (or the AI + editor pair).
> One brief = one cornerstone piece, published in weeks 1–4 (one per week, plus one held in reserve).

---

## Cornerstone 1 — "My free-tier AI stack: what I actually run, on a Tuesday"

- **Slug:** `/free-tier-ai-stack`
- **Primary intent:** "free AI tools for productivity" — informational, comparison.
- **Target query phrasings:** free AI tools 2026; free tier LLM stack; Gemini Groq OpenRouter free
- **Length:** 2,200–2,800 words.
- **Angle:** A working operator's actual stack, exact monthly cost (£), what each tool replaced.
- **Structure:**
  1. TL;DR table (tool, role, cost, "what it replaced").
  2. Methodology (how the tests were run, over how long, what "saves time" means here).
  3. Tool-by-tool: Gemini 2.5 Pro, Groq, OpenRouter, Cursor / Claude Code, Notion, a transcription tool, a calendar AI.
  4. What I dropped and why.
  5. What I'd add next if I had £20/month more.
- **Affiliate placement:** beehiiv, Notion, 1Password, Voibe (after first-link disclosure).
- **Internal links:** to Cornerstone 2, 3.
- **Image plan:** one stack diagram (original), 3 screenshot crops.
- **Compliance hits:** AI-augmentation label, affiliate inline disclosure, ToS check on every tool linked.

## Cornerstone 2 — "Claude Code vs Cursor for shipping side projects: 6 weeks, 12 commits"

- **Slug:** `/claude-code-vs-cursor`
- **Primary intent:** comparison, decision-aid.
- **Length:** 1,800–2,400 words.
- **Angle:** 6-week head-to-head, real PRs, real merge times, real frustrations.
- **Structure:**
  1. Verdict in the first 80 words.
  2. What I built (be specific — a small CRUD app, a CLI tool, etc.).
  3. Metric table (PRs/week, mean review time, friction events).
  4. Where each won.
  5. The honest "I'd go back to the other if…" footnote.
- **Affiliate placement:** Cursor (if program exists), Anthropic (no affiliate; disclose anyway).
- **Compliance:** AI label; "this is one operator's anecdote, not a benchmark" caveat.

## Cornerstone 3 — "Email-to-newsletter pipeline I built with free AI in a weekend (with the prompts)"

- **Slug:** `/newsletter-pipeline-free-ai`
- **Primary intent:** how-to, builder.
- **Length:** 2,000–2,400 words.
- **Angle:** Concrete pipeline (inbox → cluster → draft → human edit → send), with the prompts in a copy-pasteable code block.
- **Structure:**
  1. The output (screenshot of the resulting newsletter section).
  2. The pipeline diagram.
  3. The prompts (clearly labelled, code-fenced).
  4. The trade-offs (what it does badly).
  5. The £/month cost.
- **Affiliate placement:** beehiiv, Kit.
- **Compliance:** AI label; reminder that prompts are starting points, not magic.

## Cornerstone 4 — "The 5 AI workflows I cut this quarter (and what replaced them)"

- **Slug:** `/ai-workflows-cut`
- **Primary intent:** contrarian, durable.
- **Length:** 1,400–1,800 words.
- **Angle:** Anti-tool-listicle listicle — what stopped working and why.
- **Structure:** 5 short cases, each ~250 words, with a "what replaced it" line.
- **Affiliate placement:** sparing; one or two genuine recommendations.

## Cornerstone 5 — "How I review a PR in 6 minutes with AI (without being the AI's hostage)"

- **Slug:** `/pr-review-with-ai`
- **Primary intent:** how-to, engineer audience.
- **Length:** 1,400–1,800 words.
- **Angle:** A practical workflow that keeps the engineer in charge.
- **Structure:**
  1. The 6-minute checklist.
  2. The exact prompts.
  3. What AI never gets right and how to spot it.
  4. The "don't ship if…" hard rules.
- **Affiliate placement:** Claude Code / Cursor; 1Password (security check tie-in).
- **Compliance:** AI label.

---

## Editorial standards for every cornerstone

- TL;DR within the first 100 words.
- One original diagram, screenshot, or photo.
- One contrarian / specific opinion the rest of the SERP doesn't have.
- ≥3 internal links (forward to upcoming pieces; backward to existing pieces).
- Author byline = real operator name + photo.
- AI-augmentation label above the byline.
- Affiliate inline disclosure above the first affiliate link.

# Dashboard Blueprint

## Purpose
The dashboard is the command center for HamMediaLabs.

## Minimum viable dashboard
### Must include:
- Portfolio overview
- Brand health
- Content performance
- Agent health
- Provider health
- Kill / hold / scale queue

## Data sources
- CMS analytics
- Social APIs
- Provider usage
- Internal task logs
- Manual notes

## Future goals
- Predictive scoring
- Opportunity scoring
- Automated recommendations

## Rule
If it cannot be measured, it should not be scaled.
